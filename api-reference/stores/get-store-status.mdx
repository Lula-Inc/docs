---
title: "Get Store Status"
api: "GET https://api-staging.luladelivery.store/stores/{store_id}/status?current_local_date={date}"
description: "This endpoint retrieves the current operational status of a store across all delivery service partners (UberEats, DoorDash, GrubHub). It provides real-time information about whether the store is accepting orders, any active pauses, and detailed status descriptions for each platform."
playground : "simple"
---

This endpoint provides comprehensive store status information across all integrated delivery service partners. It shows whether the store is currently accepting orders, any temporary pauses, and platform-specific status details.

### Path Parameters

<ParamField path="store_id" type="string" required>
    The unique identifier of the store to check status for
</ParamField>

### Query Parameters

<ParamField query="current_local_date" type="string" required>
    Current local date and time for the store's timezone. Format: "MMM DD YYYY HH:mm:ss"
    
    Example: "Nov 15 2024 03:46:34"
</ParamField>

### Response

<ResponseField name="current_pause" type="string">
    Any current pause reason affecting all platforms
</ResponseField>

<ResponseField name="is_close" type="boolean">
    Overall store closure status across all platforms
</ResponseField>

<ResponseField name="is_closed" type="boolean">

</ResponseField>

### Status Interpretation

<Accordion title="Understanding Store Status">
    <ResponseField name="Overall Status" type="interpretation">
        `is_close` and `is_closed` indicate if the store is closed across all platforms
        
        <Info>A store can be open on some platforms and closed on others</Info>
    </ResponseField>
    
    <ResponseField name="Platform Status" type="interpretation">
        Each platform (UberEats, DoorDash, GrubHub) has independent status
        
        <Note>ONLINE = accepting orders, OFFLINE = not accepting orders</Note>
    </ResponseField>
    
    <ResponseField name="End Times" type="interpretation">
        `end_time` indicates when the current status will automatically change
        
        <Tip>null end_time means the status is permanent until manually changed</Tip>
    </ResponseField>
    
    <ResponseField name="Pause Status" type="interpretation">
        `current_pause` shows temporary holds on operations
        
        <Warning>Pauses can be platform-specific or affect all platforms</Warning>
    </ResponseField>
</Accordion>

**Real-time Data:** Status information is fetched in real-time from each delivery partner's systems, providing the most current operational state.
</Info>

<Note>
**Timezone Consideration:** The current_local_date parameter should reflect the store's local timezone for accurate status reporting, especially for scheduled operations.
</Note>
